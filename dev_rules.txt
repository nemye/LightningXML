The following are critical instructions and guidelines for all responses:

We primarily develop C++ projects using up to the C++20 standard. I prefer to lean into modern C++ including templates, and STL features like concepts, ranges, etc. C style syntax is extremely frowned upon.

1. Use the provided clang-tidy file for syntax style. Clean code principles are to be followed. 
2. Be as concise as possible. I prefer to only develop what is needed following YAGNI (you aren't going to need it) principle
3. Code comments should be limited to only explaining "why" when absolutely necessary. Superflous "what" comments should be avoided at all costs. I believe that comments are a code smell, and I would really like them to be limited. 
4. Doxygen comments should be used in the C++ stream style for functions class and templates as shown below:
    /// @brief Short description.
    /// @param t short description for t
5. Performance, maintainability, safety, and robust implementations with limited complexity are preferred. Re-use as functionality in the library as possible to avoid duplicate code or replicated behaviors. 
6. Third party dependencies are to be avoided.
7. Do not use any variations of comment section breaks common to AI code generation such as repeated "--" or "##". Avoid words like "genuinily, canonical, lexical" and other GPT-like phrasings. 
8. Prefer the use of {} initializers over = assignment. 
9. Use trailing return types.
10. Compile time work with constexpr and const correctness are important to leverage. 

In general, prefer modern C++ (up to std 20) approaches.
